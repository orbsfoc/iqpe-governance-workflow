# Diagram Drift Protocol Template

Use as `docs/diagrams/drift-protocol.md` (or draft counterpart path).

## Scope
- Diagram set covered:
- Owner:

## Drift triggers
- Plan/task ID changes (`PLAN-*`)
- Contract/interface changes
- Topology changes
- Error-path changes

## Update rules
1. Identify impacted `DIAG-*` elements.
2. Update diagram source `.mmd` files.
3. Re-validate Mermaid syntax.
4. Update mapping table and traceability links.
5. Record drift event in change log.

## Verification
- [ ] Mermaid validation passed
- [ ] Mapping table updated
- [ ] Traceability matrix updated
